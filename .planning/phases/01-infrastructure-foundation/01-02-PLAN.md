<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → nginx :443 | All external traffic terminates TLS at nginx; only /api and /_ reach PocketBase |
| public internet → port 8090 | Must be CLOSED; direct access bypasses nginx and all rate/TLS controls |
| nginx → PocketBase 127.0.0.1:8090 | Loopback-only; not routable from outside the host |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Elevation of Privilege | PocketBase port 8090 | mitigate | Bind `--http=127.0.0.1:8090` (loopback only) + firewall `ufw deny 8090`; Task 3 verifies external curl to :8090 times out (high-severity gate) |
| T-01-05 | Information Disclosure | nginx location ordering | mitigate | `/api/` and `/_/` blocks before `location /`; verify `/api/...` returns JSON 404 not SPA HTML (Pitfall 2) |
| T-01-06 | Elevation of Privilege | PocketBase admin UI `/_/` | mitigate | Superuser created on first launch with strong password; `/_/` only reachable via HTTPS through nginx, never via raw port |
| T-01-07 | Spoofing / MITM | HTTPS for andescode.com.ar | mitigate | Existing Let's Encrypt cert confirmed in Task 1 audit (INFRA-03 / D-01); all API traffic over TLS |
| T-01-SC | Tampering | PocketBase binary download | mitigate | Download only from official github.com/pocketbase/pocketbase/releases; verify version tag before pinning URL |
</threat_model>
