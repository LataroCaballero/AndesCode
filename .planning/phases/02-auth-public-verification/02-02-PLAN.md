<objective>
Deliver the public certificate verification vertical slice end-to-end — the CORE VALUE of the product. A `/certificados` search page normalizes any-case/no-dash IDs and routes to the detail page; `/certificados/:certificateCode` fetches the certificate from PocketBase (using the public `viewRule = ""` — no auth) and renders a prominent VALID/REVOKED status badge above the fold on mobile, the student name, a full metadata grid, technologies as tags, a copy-URL button, and a clean "no encontrado" state for unknown codes.

After this plan, a real user can: open a certificate URL from a QR scan and instantly see its validity, or type an ID into the search box (in any format) and reach the right certificate — all with no account and no login. This consumes the `pb` client established in Plan 02-01.

Purpose: VERIF-01 through VERIF-09, VIS-02/04/05 — anyone verifies a certificate's authenticity in seconds, mobile-first, no friction.
Output: `src/types/certificate.ts`, `src/sections/CertificadosSearch.tsx`, `src/pages/certificados.tsx`, `src/sections/CertificadoVerificacion.tsx`, `src/pages/certificado.tsx`, updated `src/main.tsx` routing, updated `src/components/TitleManager.tsx`.
</objective>
