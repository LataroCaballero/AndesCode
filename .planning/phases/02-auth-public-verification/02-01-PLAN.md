<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → PocketBase `/api/collections/_superusers/auth-with-password` | Untrusted credentials submitted from the public login form |
| localStorage ↔ app | Auth token persisted client-side; readable by any script running on the origin |
| client route guard → protected content | AdminGuard is the only thing stopping a guest from rendering /admin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | /admin/login auth | mitigate | Authentication delegated to PocketBase `_superusers.authWithPassword` (server-side bcrypt verify); no custom credential check; wrong creds yield generic error (no user-enumeration leak) |
| T-02-02 | Elevation of Privilege | client-side AdminGuard | mitigate | Client guard is UX only; REAL access control is the collection API rules from Phase 1 (`createRule/updateRule/deleteRule = null`) enforced server-side. Guest who bypasses the guard still gets 403 on any write. AdminGuard documented as defense-in-depth, not the security boundary |
| T-02-03 | Spoofing / Stale session | reload session | mitigate | AdminGuard calls `authRefresh()` on mount (AUTH-03) — an expired/revoked token is rejected server-side and the user is redirected to login; isValid alone is not trusted |
| T-02-04 | Information Disclosure | token in localStorage | accept | PocketBase LocalAuthStore default; XSS would be required to exfiltrate. No third-party scripts handle the token; same-origin nginx proxy means no cross-origin token exposure. Single-admin system, low blast radius |
| T-02-05 | Session fixation | logout | mitigate | `pb.authStore.clear()` removes token + record from store and localStorage; redirect uses `replace:true` to prevent back-button re-entry into protected view |
| T-02-SC | Tampering | npm installs | accept | No new packages installed; `pocketbase@^0.27.0` already in package.json (slopcheck `[OK]` per Phase 1 Package Legitimacy Audit). No package-manager install task in this plan |
</threat_model>
