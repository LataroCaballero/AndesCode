| Token | Value | Usage |

| sm | 8px | Button internal padding (vertical), tag gaps |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `var(--color-ink)` text on white | Page backgrounds, card backgrounds, login form background, verification page background |
| Secondary (30%) | `#F9FAFB` (gray-50) / `#191919` surfaces | Grid background (`.grid-bg`), metadata grid cells on verification page, admin container background |
| Accent (10%) | `#4342FF` (`var(--color-primary)`) | See reserved list below |
| Valid status | `#16A34A` (green-600) | VALID status badge background and text only |
| Revoked/Destructive | `#FF6829` (`var(--color-warning)`) | REVOKED status badge background and text only |
| Not Found | `#6B7280` (gray-500) | "Certificado no encontrado" state illustration/icon |

**Accent (`#4342FF`) reserved for:**
1. Primary CTA button fill (`btn-primary`) — "Verificar certificado" and "Iniciar sesión"
2. Certificate code text displayed on verification page (monospace emphasis)
3. Section label badges (uppercase tracking-widest labels, e.g. "Verificación pública")
4. Input focus ring (`outline-[var(--color-primary)]`)
5. Search field submit icon/arrow button

Accent is NOT used for: body text, metadata labels, status badges, card borders.

**Status badge color contract (critical — VIS-05):**
- VALID: green background `bg-green-100`, green text `text-green-700`, green border `border-green-200` — label: "Certificado válido"
- REVOKED: orange/red background `bg-red-100`, red text `text-red-700`, border `border-red-200` — label: "Certificado revocado"
- NOT FOUND: gray background `bg-gray-100`, gray text `text-gray-600` — displayed on `/certificados` search result inline (NOT on a certificate detail page)

These three states must be visually distinct with no color overlap.

Source: `src/style.css` CSS custom properties; existing green/red Tailwind conventions in codebase.

## Interaction Contracts

### `/certificados` — Search Page

**Input field:**
- Placeholder: `AC-AAAA-NNN` (format hint, Fira Code Regular)
- `type="text"` with `autocomplete="off"`, `spellcheck="false"`
- On change: no real-time normalization (normalize only on submit)
- On submit: normalize input (uppercase, insert dashes if absent), navigate to `/certificados/{normalizedCode}`
- Min-height: 48px (mobile touch target)
- Focus ring: `outline outline-2 outline-[var(--color-primary)]`

**Submit button:**
- Label: "Verificar certificado"
- Style: `btn-primary`, full-width on mobile, auto-width on desktop
- Min-height: 48px
- Loading state: spinner replaces label text during normalization (if async lookup needed)

**ID normalization rules (VERIF-02):**
1. Trim whitespace
2. Uppercase all characters
3. Remove all dashes, then reinsert as `XX-XXXX-XXX` (first 2 chars dash next 4 chars dash rest)
4. Navigate to `/certificados/{result}`

**"Not found" inline state (VERIF-03):**
- Displayed as inline message below the input — NOT a separate page
- Copy: "No encontramos un certificado con ese ID. Revisá el código e intentá de nuevo."
- Color: gray (`text-gray-600`, `bg-gray-50` background strip)
- Icon: `FiAlertCircle` (react-icons/fi) — gray
- Visually distinct from REVOKED (which is orange/red and shows on the certificate detail page)

### `/certificados/:certificateCode` — Verification Page

**Loading state:**
- Full section shows skeleton placeholders (gray animated pulse blocks) while fetching
- Status badge area is the first skeleton rendered — same position it occupies when loaded

**Status badge (VIS-02, VIS-05 — must be above fold on mobile):**
- Renders as the FIRST element inside the section, above all other certificate data
- VALID: `bg-green-100 border border-green-200 text-green-700` pill badge, 48px min-height, full-width on mobile
  - Icon: `FiCheckCircle` (react-icons/fi), size 20px
  - Text: "Certificado válido emitido por AndesCode"
  - Font: Inter semibold 16px
- REVOKED: `bg-red-100 border border-red-200 text-red-700` pill badge, same dimensions
  - Icon: `FiXCircle` (react-icons/fi), size 20px
  - Text: "Certificado revocado"
  - Font: Inter semibold 16px
- Both badges span full width on mobile, max-width 480px centered on desktop

**Student name (Display typography):**
- Rendered as `<h1>`: Inter Bold, 28px mobile / 36px desktop, `text-[#191919]`
- All caps: `uppercase` (matches physical certificate visual)
- Position: immediately below the status badge

**Certificate metadata grid:**
- Displayed as a 2-column grid on mobile, 3-column on tablet+
- Each cell: label (Inter 12px uppercase tracking-widest gray-500) + value (Inter 14px `text-[#191919]`)
- Fields: ID del certificado (Fira Code Regular), Fecha de emisión, Período de la práctica (start–end), Universidad, Área de desempeño, Calificación, Supervisor
- Certificate ID cell: Fira Code Regular 14px, `text-[var(--color-primary)]`

**Description block:**
- Label: "Descripción de la práctica" (Inter 12px uppercase tracking-widest gray-500)
- Body: Inter Regular 14px, `text-gray-700`, max-width 640px

**Technology tags (VERIF-07):**
- Each tag: `bg-[#4342FF]/10 text-[#4342FF] text-xs fira-code-regular px-3 py-1 rounded-full`
- Layout: flex-wrap gap-2
- Label above: "Tecnologías utilizadas" (Inter 12px uppercase tracking-widest gray-500)

**Copy-URL button (VERIF-08):**
- Label (default): "Copiar enlace de verificación"
- Label (after copy, 2s): "Enlace copiado!"
- Style: `btn-secondary` with `FiCopy` icon (react-icons/fi) left of text
- Behavior: `navigator.clipboard.writeText(window.location.href)`, then show success state for 2000ms
- Position: below the metadata grid, above technologies section

**Not found state (when navigating directly to an unknown code):**
- Replaces the entire certificate content area
- Heading: "Certificado no encontrado"
- Body: "No existe un certificado con el código [CODE]. Verificá que el ID sea correcto."
- CTA link: "Buscar otro certificado" → navigates to `/certificados`
- Style: centered, gray illustration area, same section height as normal certificate

### `/admin/login` — Login Page

**Layout:** Standalone page — no `Header`, no `Footer`. Centered card on full-viewport background (`.grid-bg` pattern matching site aesthetic).

**Form card:**
- Background: white, `rounded-xl`, `shadow-[0_4px_24px_rgba(67,66,255,0.10)]`
- Padding: 32px (lg token)
- Max-width: 400px, centered horizontally and vertically in viewport

**Heading:**
- "ANDESCODE" — Fira Code Bold, 20px, `text-[#191919]` (matches site logo style)
- Subheading: "Panel de administración" — Inter Regular, 14px, `text-gray-500`

**Email input:**
- Label: "Correo electrónico"
- Type: `email`, `autocomplete="email"`
- Min-height: 44px
- Focus ring: `outline outline-2 outline-[var(--color-primary)]`

**Password input:**
- Label: "Contraseña"
- Type: `password`, `autocomplete="current-password"`
- No "show password" toggle in Phase 2
- Min-height: 44px

**Submit button:**
- Label: "Iniciar sesión"
- Style: `btn-primary`, full-width
- Loading state: disabled + spinner while awaiting PocketBase response
- Min-height: 44px

**Error state:**
- Displayed as a single inline alert above the submit button
- Copy: "Credenciales incorrectas. Revisá tu correo y contraseña."
- Style: `bg-red-50 border border-red-200 text-red-700 rounded-lg px-4 py-3 text-sm`
- Icon: `FiAlertCircle` (react-icons/fi)
- Appears after failed login attempt; disappears when user modifies either input field

**Success behavior:** On successful login, navigate to `/admin` immediately (no success toast — navigation is the confirmation).

### `/admin` — Protected Dashboard

**Access control (AUTH-05):**
- `AdminGuard` wraps all `/admin/*` routes
- On mount: read `pb.authStore.isValid` — if false, `navigate('/admin/login', { replace: true })`
- On token revalidation (AUTH-03): call `pb.collection('users').authRefresh()` on mount; if it throws, clear store and redirect
- Redirect uses `replace: true` to prevent back-button returning to the protected page

**Dashboard placeholder (Phase 2 scope only):**
- Shows admin's email address: "Bienvenido, {email}"
- Logout button: "Cerrar sesión" — `btn-secondary`, calls `pb.authStore.clear()`, navigates to `/admin/login`
- No sidebar, no nav — Phase 3 will introduce the admin chrome

## Mobile-First Layout Directives (VIS-04)

The verification page is the primary mobile use case (QR scans from phone). These rules are non-negotiable:

1. **Status badge above fold:** The status badge must render in the first 400px of vertical page content on a 390px-wide screen (iPhone 14 baseline). This means: no hero image, no large top padding before the badge. Section `pt-` must be ≤ 96px (accounting for 80px fixed header).

2. **Touch targets:** All interactive elements (buttons, copy link) must be ≥ 44px tall.

3. **Technology tags:** Wrap freely — no horizontal scroll. `flex-wrap gap-2`.

4. **Metadata grid:** 2 columns on mobile (≤ 767px), 3 columns on tablet+.

5. **Student name:** Wraps naturally — no `truncate` or `nowrap` — can be 2 lines on narrow screens.

6. **Search input + button:** Stacked vertically on mobile (flex-col), side-by-side on sm+ (flex-row sm:flex-row).
