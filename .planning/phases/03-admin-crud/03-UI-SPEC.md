# Phase 3 — UI Design Contract: Admin CRUD

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> All copy is in Spanish. Site language: es-AR.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual Tailwind CSS v4 + CSS custom properties) |
| Preset | not applicable |
| Component library | none — custom components only |
| Icon library | react-icons v5 (fi family: `FiEdit2`, `FiToggleLeft`, `FiToggleRight`, `FiDownload`, `FiX`, `FiPlus`, `FiSearch`, `FiAlertCircle`, `FiChevronLeft`, `FiChevronRight`) |
| Font (body) | Inter (Google Fonts, weights 400 and 600) |
| Font (display/mono) | Fira Code (Google Fonts, utility classes `.fira-code-bold`, `.fira-code-regular` in `src/style.css`) |

Source: Phase 2 UI-SPEC + codebase scan — `src/style.css`, `src/pages/admin/login.tsx`, `package.json`

## Routes in Scope

| Route | Layout | Auth Required |
|-------|--------|---------------|
| `/admin` | Admin shell (no public Header/Footer) — top bar sticky + full-page content area | Yes — AdminGuard redirects to `/admin/login` |

This phase introduces no new routes. All admin CRUD functionality lives in `/admin` via local React state (drawer open/closed, modal open/closed). The public routes (`/certificados`, `/certificados/:certificateCode`, `/admin/login`) are untouched in Phase 3.

The admin shell does NOT use the public `<Header />` or `<Footer />` components. It shares no visual chrome with the marketing site. Background: `.grid-bg` (same as login page — D-02).

## Spacing Scale

Identical to Phase 2 contract (inherited — source: 02-UI-SPEC.md):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline tag padding, action button icon gaps |
| sm | 8px | Button internal padding (vertical), tag gaps, table cell vertical padding |
| md | 16px | Default element spacing, card internal padding (horizontal), form field gaps |
| lg | 24px | Section internal padding, drawer header padding, modal padding |
| xl | 32px | Layout gaps between content blocks, drawer content padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets: minimum 44px height for all interactive elements (`min-h-[44px]`) — applies to all table action buttons, form inputs, filter controls, pagination buttons
- Top bar height: 56px (`h-14`) — compact sticky bar; taller than input height but shorter than public Header (80px)
- Drawer width: `max-w-[600px]` on desktop, full-width on mobile — fixed by D-03
- Modal max-width: `max-w-[440px]` — fits confirmation content without overflow on mobile

Source: Phase 2 UI-SPEC (inherited); top bar height and modal width determined at researcher discretion (D in CONTEXT.md).

## Color

Identical tokens to Phase 2 (source: 02-UI-SPEC.md + `src/style.css`):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `var(--color-ink)` text on white | Page background (`.grid-bg` with white base), drawer panel background, modal background, table row background |
| Secondary (30%) | `#F9FAFB` (gray-50) / `#E5E7EB` (gray-200) | Table header row background, alternate row tint (optional), drawer backdrop overlay, form section dividers |
| Accent (10%) | `#4342FF` (`var(--color-primary)`) | See reserved list below |
| Destructive | `#b91c1c` (red-700) | Revoke confirmation modal primary button ONLY |
| Reactivate | `#16A34A` (green-600) | Reactivate confirmation modal primary button ONLY |

**Accent (`#4342FF`) reserved for (Phase 3 additions):**
1. Primary CTA button fill (`btn-primary`) — "Guardar certificado", "Nuevo certificado"
2. Certificate code text in table rows (monospace emphasis)
3. Input focus ring (`outline-[var(--color-primary)]`) — all form inputs
4. Tag chip background/text in TagsInput: `bg-[#4342FF]/10 text-[#4342FF]`
5. Top bar accent line (optional 2px bottom border — researcher discretion, see top bar spec below)

Accent is NOT used for: table row text, metadata labels, status badges, table borders, pagination numbers, modal overlays.

**Status badge color contract (table rows — must be visually distinct):**
- ACTIVE: `bg-green-100 text-green-700 border border-green-200` — label: "Activo"
- REVOKED: `bg-red-100 text-red-700 border border-red-200` — label: "Revocado"

These two states must be distinguishable at a glance in the table. No other status values exist in v1.

Source: `src/style.css` CSS custom properties; Phase 2 UI-SPEC (inherited); login.tsx for error state pattern.

## Interaction Contracts

### `/admin` — Top Bar (`AdminTopBar`)

**Layout:** Fixed to top of viewport; `position: sticky; top: 0; z-index: 30`. Height: 56px (`h-14`). Background: white (`bg-white`). Bottom border: `border-b border-gray-200`.

**Contents (left to right):**
1. Wordmark: "ANDESCODE" — `fira-code-bold text-xl text-[#191919]`
2. Spacer: `flex-1`
3. Admin identity: `{record.email}` — Inter Regular 14px, `text-gray-500`; hidden on screens below 480px (`hidden sm:block`)
4. Logout button: "Cerrar sesión" — `btn-secondary`, `min-h-[44px] px-4 text-sm rounded-lg`; calls `pb.authStore.clear()` + `navigate('/admin/login', { replace: true })`

**Behavior:** No confirmation on logout — single-click (established in Phase 2; same as existing placeholder).

### `/admin` — Certificate List (`AdminCertificateList`)

**Layout:** Full-width content area below the top bar. Padding: `px-4 sm:px-8 py-6`. Background: `.grid-bg` (full page background from `AdminPage` wrapper).

**Controls row (above table):**

- **Search input:**
  - Placeholder: "Buscar por nombre o código…"
  - Type: `text`, width: `max-w-[320px]` on desktop, full-width on mobile
  - Icon: `FiSearch` (react-icons/fi), 16px, left-aligned inside input, `text-gray-400`
  - Debounce: 300ms before triggering PocketBase query
  - Min-height: 44px
  - Border: `border border-gray-300 rounded-lg`
  - Focus: `focus:outline focus:outline-2 focus:outline-[var(--color-primary)]`

- **Status filter select:**
  - Options: "Todos", "Activo", "Revocado" — values: `all`, `active`, `revoked`
  - Style: `border border-gray-300 rounded-lg px-3 py-2 text-sm min-h-[44px]`
  - Focus: `focus:outline focus:outline-2 focus:outline-[var(--color-primary)]`
  - Default: "Todos"

- **"Nuevo certificado" button:**
  - Style: `btn-primary min-h-[44px] px-4 text-sm rounded-lg flex items-center gap-2`
  - Icon: `FiPlus` (react-icons/fi), 16px
  - Position: right-aligned in controls row
  - Action: opens drawer in create mode (calls `openDrawer('create')`)

**Table:**

Columns (in order):

| Column | Width | Content |
|--------|-------|---------|
| Código | 140px | `certificateCode` — `fira-code-regular text-[var(--color-primary)]` |
| Nombre del estudiante | flex / min-w-0 | `studentName` — Inter Regular 14px; truncate with ellipsis if overflow |
| Fecha de emisión | 130px | `issueDate` formatted as `DD/MM/YYYY` — Inter Regular 14px |
| Estado | 100px | Status badge pill (see color contract) |
| Acciones | 120px | 3 icon buttons — always visible (D-09, deferred rule) |

**Table header row:**
- Background: `bg-gray-50`
- Text: Inter Regular 14px uppercase tracking-widest `text-gray-500`
- Border: `border-b border-gray-200`

**Table body rows:**
- Background: white, `hover:bg-gray-50` transition
- Border: `border-b border-gray-100`
- Min-height per row: 52px

**Action buttons per row (3 icons, always visible — D-09):**
1. Edit: `FiEdit2`, 16px — opens drawer in edit mode with this record pre-populated
2. Revoke/Reactivate toggle:
   - If `status === 'active'`: `FiToggleLeft`, 16px, `text-gray-400` — opens revoke confirm modal
   - If `status === 'revoked'`: `FiToggleRight`, 16px, `text-green-600` — opens reactivate confirm modal
3. Download QR: `FiDownload`, 16px — triggers SVG download for this certificate's QR code

All action buttons:
- Container: `min-h-[44px] min-w-[44px] flex items-center justify-center`
- Style: icon-only buttons with `title` attribute (tooltip on hover) and `aria-label`
- Hover: `hover:text-[var(--color-primary)]` or action-appropriate color
- Gap between action buttons: 4px (xs token)

**Pagination:**
- Position: below the table, right-aligned row
- Controls: "← Anterior" + page info (`Página {N} de {M}`) + "Siguiente →"
- Style: `btn-secondary` for prev/next; Inter 14px gray for page info; both buttons disabled when at boundary
- Items per page: 20 (constant — not user-configurable in v1; Claude's discretion from CONTEXT.md)
- Default sort: `-created` (most recent first — Claude's discretion from CONTEXT.md)

**Empty state (no results):**
- Displayed in place of table body when `items.length === 0`
- Heading: "No hay certificados"
- Body: "Creá el primer certificado con el botón de arriba." (when no filter active) — or — "No encontramos certificados con ese criterio. Modificá la búsqueda o el filtro." (when search or filter is active)
- Icon: `FiSearch` (react-icons/fi), 32px, `text-gray-300`
- No table headers shown in empty state

**Loading state:**
- 5 skeleton rows replace the table body: animated `animate-pulse` gray bars for each column
- Table header is visible during loading

**Error state (fetch failed):**
- Displayed in place of table body
- Copy: "Error al cargar los certificados. Revisá tu conexión e intentá de nuevo."
- Style: `bg-red-50 border border-red-200 text-red-700 rounded-lg px-4 py-3 text-sm flex items-center gap-2`
- Icon: `FiAlertCircle`, 16px
- Retry: includes a "Reintentar" text button (inline, `text-[var(--color-primary)]`)

### `/admin` — Create/Edit Drawer (`AdminCertificateDrawer` + `AdminCertificateForm`)

**Drawer shell:**
- Framer-motion `AnimatePresence` wrapper (import from `'framer-motion'` — NOT `'motion/react'`)
- Backdrop: `fixed inset-0 bg-black/30 z-40` — click to close drawer
- Panel: `fixed right-0 top-0 h-full w-full max-w-[600px] bg-white shadow-2xl z-50 overflow-y-auto`
- Enter: `initial={{ x: '100%' }} animate={{ x: 0 }}` — `type: 'tween', duration: 0.2, ease: 'easeOut'`
- Exit: `exit={{ x: '100%' }}` — same timing
- Backdrop enter/exit: `opacity 0 → 1`, duration 0.2s
- Scroll lock: `document.body.style.overflow = 'hidden'` when open; restored on close via `useEffect` cleanup

**Drawer header:**
- Height: 56px, `border-b border-gray-200 px-6 flex items-center justify-between`
- Title: "Nuevo certificado" (create mode) or "Editar certificado" (edit mode) — Inter Semibold 20px `text-[#191919]`
- Close button: `FiX` icon, 20px, `min-h-[44px] min-w-[44px] flex items-center justify-center` — closes drawer

**Form layout (inside drawer, `AdminCertificateForm`):**

Two-column grid on desktop (drawer ≥ 480px), single column on mobile:
- Grid: `grid grid-cols-1 sm:grid-cols-2 gap-x-6 gap-y-5 px-6 py-6`

Field column assignment:

Left column (personal info):
1. `studentName` — Text input, required — label: "Nombre completo del estudiante"
2. `dni` — Text input, required — label: "DNI"
3. `university` — Text input, required — label: "Universidad / Facultad"
4. `degree` — Text input, required — label: "Área de desempeño"
5. `supervisorName` — Text input, required — label: "Supervisor"
6. `score` — Number input, optional — label: "Calificación"; `min=0 max=10 step=0.1`

Right column (dates + code):
1. `certificateCode` — Text input, required — label: "Código del certificado"; `fira-code-regular`; pre-populated in create mode (format `AC-YYYY-NNN`); editable
2. `issueDate` — Date input, required — label: "Fecha de emisión"
3. `startDate` — Date input, required — label: "Inicio de la práctica"
4. `endDate` — Date input, required — label: "Fin de la práctica"

Full-width (span 2 columns):
1. `description` — Textarea, optional, min-height 80px — label: "Descripción de la práctica"
2. `technologies` — `TagsInput`, optional — label: "Tecnologías utilizadas"; placeholder: "Escribí una tecnología y presioná Enter"
3. `competencies` — `TagsInput`, optional — label: "Competencias"; placeholder: "Escribí una competencia y presioná Enter"

The `status` field is NOT in the form. Status changes are made exclusively via the revoke/reactivate buttons in the list (D-07). Including status in the form would bypass the confirmation modal (D-08).

**All form inputs (text, date, number):**
- Container: `flex flex-col gap-1.5`
- Label: `text-sm font-medium text-[#191919]`
- Input: `border border-gray-300 rounded-lg px-4 py-2.5 text-sm focus:outline focus:outline-2 focus:outline-[var(--color-primary)] transition min-h-[44px] w-full`
- Error state (validation failed on submit): `border-red-400` + red helper text below input: `text-red-600 text-xs mt-1`

**Required field indicator:** Red asterisk `*` after label text for required fields — `text-red-500 ml-0.5`

**TagsInput component:**
- Wrapper: `border border-gray-300 rounded-lg px-3 py-2 flex flex-wrap gap-1.5 focus-within:outline focus-within:outline-2 focus-within:outline-[var(--color-primary)] min-h-[44px]`
- Tag chip: `bg-[#4342FF]/10 text-[#4342FF] text-sm fira-code-regular px-2 py-0.5 rounded-full flex items-center gap-1`
- Remove button on chip: `×` character, `text-[#4342FF] hover:text-[#4342FF]/70`; `aria-label="Eliminar {tag}"`
- Text input inside: `flex-1 min-w-[120px] outline-none text-sm bg-transparent`
- Enter or comma key: adds current input as a tag (trim whitespace; skip duplicates; clear input)
- Backspace when input empty: removes last tag

**Drawer footer:**
- `border-t border-gray-200 px-6 py-4 flex justify-end gap-3`
- Cancel button: "Cancelar" — `btn-secondary min-h-[44px] px-6 text-sm rounded-lg` — closes drawer without saving
- Save button: "Guardar certificado" — `btn-primary min-h-[44px] px-6 text-sm rounded-lg` — triggers client-side validation then PocketBase create/update
- Loading state (saving): Save button disabled + spinner; label: "Guardando…"; opacity 0.7

**Validation on submit:**
- All required fields must be non-empty strings (or valid dates/numbers)
- If any required field is empty: mark that field with `border-red-400` + helper text; scroll drawer to first error; do NOT call PocketBase
- `certificateCode` uniqueness: not validated client-side; PocketBase 400 error `data.certificateCode.code === 'validation_not_unique'` shown as drawer-level error: "El código ya existe. Modificá el código e intentá de nuevo."
- General save error: "Error al guardar. Intentá de nuevo." — displayed as inline alert above the footer buttons (same `bg-red-50 border border-red-200 text-red-700 rounded-lg` pattern as login.tsx)

### `/admin` — Confirmation Modal (`ConfirmModal`)

Used for both revoke (D-07) and reactivate (D-10) actions.

**Layout:**
- Backdrop: `fixed inset-0 bg-black/40 z-50 flex items-center justify-center px-4`
- Card: `bg-white rounded-xl shadow-2xl w-full max-w-[440px] p-6`

**Revoke modal content:**
- Heading: "¿Revocar este certificado?" — Inter Semibold 20px `text-[#191919]`
- Body: "Vas a revocar el certificado **{certificateCode}** de **{studentName}**. Esta acción se puede revertir." — Inter Regular 14px `text-gray-600`
- Emphasis (`**...**`): `font-semibold text-[#191919]`
- Button row (`justify-end gap-3`):
  - "Cancelar" — `btn-secondary min-h-[44px] px-5 text-sm rounded-lg`
  - "Revocar" — `min-h-[44px] px-5 text-sm rounded-lg bg-red-600 text-white border border-red-600 hover:bg-red-700`

**Reactivate modal content:**
- Heading: "¿Reactivar este certificado?" — same Inter Semibold 20px
- Body: "Vas a reactivar el certificado **{certificateCode}** de **{studentName}**. Va a volver a aparecer como válido." — Inter Regular 14px `text-gray-600`
- Button row:
  - "Cancelar" — `btn-secondary min-h-[44px] px-5 text-sm rounded-lg`
  - "Reactivar" — `min-h-[44px] px-5 text-sm rounded-lg bg-green-600 text-white border border-green-600 hover:bg-green-700`

**Both modals:**
- Loading state (after confirm click): confirm button disabled + spinner; label changes to "Revocando…" or "Reactivando…"
- Success: modal closes, list re-fetches (refreshKey increment)
- Error: inline alert inside modal above button row: "Error al actualizar el estado. Intentá de nuevo." — same `bg-red-50` pattern
- Pressing Escape key closes modal (same as clicking Cancelar)

### QR SVG Download (per-row action)

Triggered by `FiDownload` button in each table row action column.

**Pattern (RESEARCH.md Pattern 5):**
- Use lazy mount: do NOT render `QRCodeSVG` for all rows simultaneously
- On download click: mount a single hidden `QRCodeSVG` off-screen, serialize, download, unmount
- QR value: `{window.location.origin}/certificados/{certificateCode}`
- QR props: `size={256}`, `level="H"`, `marginSize={4}`
- File name: `QR-{certificateCode}.svg`
- Implementation: `XMLSerializer` + `URL.createObjectURL` + `<a download>` click trigger
- `aria-label`: `"Descargar QR de {certificateCode}"`
- Dependency: `qrcode.react` must be installed (`npm install qrcode.react`) before this code runs

- Table is full-width on all screens; horizontal scroll enabled on narrow screens (`overflow-x-auto` wrapper)
- Drawer: `max-w-[600px]` on desktop; `w-full` (full-screen) on mobile — framer-motion width adapts via CSS
- Form inside drawer: 2-column grid (`sm:grid-cols-2`) on drawer ≥ 480px, 1-column below
- Action buttons in table: always visible (not hover-reveal — deferred rule in CONTEXT.md)
- Controls row: search + filter left-aligned, "Nuevo certificado" right-aligned; wraps to two rows on mobile
