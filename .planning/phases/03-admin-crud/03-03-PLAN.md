<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Row action → PocketBase status update | Admin-triggered state change crosses into persistence |
| Certificate field values → QR / modal rendering | Certificate data (incl. studentName) rendered into modal and QR value |
| Browser DOM → downloaded SVG file | Serialized SVG leaves the app as a file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-STATUS-AUTHZ | Elevation of Privilege | status update call | accept | PocketBase `updateRule` + auth (INFRA-06) enforce server-side; ConfirmModal is UX safety, not the security boundary |
| T-03-CONFIRM | Tampering | accidental revoke | mitigate | ConfirmModal requires explicit confirmation showing student name + code before any status change (D-08); revoke is reversible via reactivate (ADMIN-10) |
| T-03-XSS | Tampering/Information Disclosure | modal + row rendering | mitigate | All certificate values rendered via JSX `{value}`; QR `value` is a plain URL string passed to QRCodeSVG; no `dangerouslySetInnerHTML` |
| T-03-QRURL | Information Disclosure | QR value origin | accept | `window.location.origin` is the production domain inside the authenticated admin panel; dev-origin QR only occurs in local dev (RESEARCH Pitfall 6) — documented in code comment |
</threat_model>
