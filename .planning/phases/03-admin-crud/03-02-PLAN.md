<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser form → PocketBase create/update | Admin-supplied certificate data crosses into persistence |
| Auto-code generation → certificateCode uniqueness | Client-computed sequence may collide; DB UNIQUE index is the guard |
| Long-lived form session → token validity | Token may expire while the admin fills the form |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-VALIDATION | Tampering | create/edit form | mitigate | Client-side required-field gate blocks empty submissions (ADMIN-06); PocketBase schema constraints are the authoritative server-side validation |
| T-03-UNIQUE | Tampering | certificateCode generation | mitigate | PocketBase UNIQUE index (INFRA-05) rejects duplicates with 400; form catches `validation_not_unique` and shows actionable copy. Race acceptable for single-admin v1 (RESEARCH Pitfall 3) |
| T-03-TOKEN | Authentication bypass | save after long form fill | accept | If token expires mid-fill, PocketBase returns 401 and AdminGuard redirects to /admin/login on next guarded mount; acceptable for single-admin v1 (RESEARCH Security Domain) |
| T-03-XSS | Tampering/Information Disclosure | form field rendering | mitigate | All values rendered with JSX `{value}`; no `dangerouslySetInnerHTML` |
</threat_model>
