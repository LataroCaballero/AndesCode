## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → local install | Third-party packages enter the build (jspdf, qrcode, @types/qrcode) |
| Vite static assets → jsPDF | Self-hosted TTF + logo PNGs fetched at generation time (same-origin, no CDN) |
| Certificate data → PDF stream | Typed `Certificate` fields written via `doc.text()` (no code evaluation) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-SC | Tampering | npm installs (jspdf, qrcode, @types/qrcode) | mitigate | Blocking-human package-legitimacy checkpoint (Task 1) before install; exact version pins; never auto-approved |
| T-04-04 | Tampering | jsPDF PDF object injection | mitigate | Never call `doc.addJS()` or `doc.createAnnotation()` (CVE-2026-25755 / CVE-2026-31898); pin `jspdf@4.2.1` (patched); acceptance criterion + grep gate assert absence |
| T-04-05 | Information Disclosure | PDF content | mitigate | All PDF text comes from typed `Certificate` fields; `cert.dni` never written; no raw user-input concatenation into PDF streams |
| T-04-06 | Denial of Service | Font loading CORS in production | mitigate | Self-hosted Inter via Vite `?url` (D-06/QRPDF-05) — no Google Fonts CDN, no CORS at generation time |
| T-04-07 | Tampering | Large ArrayBuffer→base64 | accept | Chunked 8192-byte conversion avoids call-stack crash; failure is caught and surfaced as an inline error, not a security event |
</threat_model>
