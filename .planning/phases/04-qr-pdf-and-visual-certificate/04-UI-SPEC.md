## Interaction Contracts

### Actions Section

**"Descargar certificado PDF" button (QRPDF-03):**
- Style: `btn-primary`, icon `FiDownload` (react-icons/fi) left of text
- Min-height: 44px
- Default label: "Descargar certificado PDF"
- Loading state: disabled + `animate-spin` spinner replaces the `FiDownload` icon + label: "Generando PDF..."
- Loading duration: while `isGenerating === true` (async jsPDF generation + font load)
- Success: button returns to default state; browser triggers file download automatically
- Error state: if PDF generation fails — button returns to default state; show inline error message below button row: "No se pudo generar el PDF. Intentá de nuevo."
- Filename: `certificado-{cert.certificateCode}.pdf` (e.g. `certificado-AC-2025-014.pdf`) — source: CONTEXT.md specifics
- Available for both `active` and `revoked` certificates — the PDF includes the REVOCADO watermark for revoked ones

**"Copiar enlace" button (VERIF-08 — unchanged from Phase 2):**
- Style: `btn-secondary`, icon `FiCopy` (react-icons/fi) left of text
- Min-height: 44px
- Default label: "Copiar enlace"
- Success label (2000ms): "Enlace copiado!"
- Behavior: `navigator.clipboard.writeText(window.location.href)`

**Button row layout:**
- Mobile (< 640px): `flex flex-col gap-3` — stacked vertically, full width buttons
- Desktop (≥ 640px): `flex flex-row gap-3` — side by side, auto-width buttons

### Certificate Visual — QR Code (QRPDF-01, QRPDF-02)

- Library: `qrcode.react` (already installed) — use `<QRCodeSVG>` component (SVG for crisp rendering at all sizes)
- Value: `window.location.href` (full verification URL including protocol and domain)
- Size: 100px × 100px in the HTML/CSS certificate visual
- Error correction level: `H` (30% redundancy — QRPDF-02)
- Background: `#FFFFFF`
- Foreground: `#191919`
- Position: bottom-right corner of certificate visual (see layout diagram above)
- No label under QR in the HTML visual (the reference shows "VERIFICÁ LA AUTENTICIDAD" — include this label: Inter Regular 8px, uppercase, centered under QR, `text-gray-500`)
- On mobile: QR moves below supervisor block (flex-col bottom row)

### Certificate Visual — Loading State

The loading skeleton from Phase 2 is extended to accommodate the new layout:

```
[Skeleton: actions row — 48px pulse block, full width]
[Gap: 16px]
[Skeleton: button row — two 44px pulse blocks]
[Gap: 32px]
[Skeleton: certificate card — rounded-xl, ~400px height, animate-pulse]
```

The skeleton does NOT attempt to reproduce the certificate internal layout — a single large rounded block is sufficient.

### PDF Generation — jsPDF Contract

- Orientation: **landscape** (A4: 297mm × 210mm) — inferred from reference image proportions (wider than tall); CONTEXT.md marks this as Claude's discretion
- Units: mm
- Font setup:
  1. `fetch(interRegularUrl)` and `fetch(interBoldUrl)` where URLs are Vite `?url` imports of the TTF files
  2. Convert ArrayBuffer to base64 string
  3. `doc.addFileToVFS('Inter-Regular.ttf', base64Regular)`
  4. `doc.addFont('Inter-Regular.ttf', 'Inter', 'normal')`
  5. Repeat for Bold
  6. `doc.setFont('Inter')`
- QR in PDF: generated with `qrcode` package (headless, not `qrcode.react`) via `QRCode.toDataURL(url, { errorCorrectionLevel: 'H', width: 200 })` → returns PNG dataURL → `doc.addImage(dataUrl, 'PNG', x, y, 30, 30)` (30mm × 30mm)
- Page margins: 15mm all sides
- Certificate layout in PDF mirrors the HTML/CSS visual structure:
  1. AndesCode logo (PNG): top-left, 35mm × 12mm
  2. FCEFN logo (PNG): top-right, 25mm × 12mm
  3. Horizontal rule at ~25mm from top
  4. "CERTIFICADO" centered, 22pt bold
  5. Subtitle centered, 9pt regular
  6. "se certifica que" centered, 9pt italic
  7. Student name centered all-caps, 18pt bold
  8. Description paragraph, 8pt regular, wrapped
  9. Horizontal rule
  10. 4-column footer: PERÍODO | ÁREA | HERRAMIENTAS | CALIFICACIÓN
  11. Supervisor block (bottom-left): name 9pt bold + "Responsable de Gestión de Talento Humano" 8pt
  12. AndesCode seal (logo repeated): bottom-center, 20mm × 20mm
  13. QR: bottom-right, 30mm × 30mm + "VERIFICÁ LA AUTENTICIDAD" 7pt below
  14. Footer line: `// CERTIFICADO: {code}` Fira Code → use Inter Regular as fallback in PDF, left-aligned + date right-aligned, 8pt, at page bottom margin
- Revoked PDF: add diagonal "REVOCADO" text overlay in red (opacity 0.25) using `doc.setTextColor(185, 28, 28)` with `doc.setGState(new doc.GState({ opacity: 0.25 }))` centered and rotated 45° (jsPDF uses degrees counterclockwise)

Source: CONTEXT.md D-05, D-06, D-07; reference image; CONTEXT.md specifics (filename).

## Mobile-First Layout Directives

Inherits Phase 2 directives. Phase 4 additions:

1. **Actions section above fold:** Status badge + both action buttons must be visible within the first 400px of vertical page content on a 390px-wide screen (iPhone 14 baseline). Stacked buttons (flex-col) ensure this is achievable.

2. **Certificate visual below fold is expected and acceptable:** The certificate is a document — it is inherently taller than a viewport on mobile. Users scroll to see it fully. The critical information (valid/revoked status + download action) is above fold.

3. **Certificate visual on mobile:** Padding reduces to 16px; font sizes follow Tailwind's built-in responsive scale; the 4-column footer row becomes 2-column grid; QR and supervisor block stack vertically.

4. **PDF is landscape:** The PDF is designed for desktop printing/viewing. The mobile experience is centered on the HTML/CSS visual + QR scan. The download button is the bridge.

5. **Touch targets:** All interactive elements remain ≥ 44px (buttons, action row). No new interactive elements inside the certificate visual itself.
